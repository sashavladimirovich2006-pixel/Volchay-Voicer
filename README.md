# Volchay Voicer

Volchay Voicer - C++/Qt6 QML приложение для изменения голоса в реальном времени. Первая версия работает по маршруту: микрофон -> DSP-обработка -> выбранное аудиоустройство вывода.

## Правила проекта

1. Каждый шаг разработки нужно записывать в этот README подробно: что сделано, зачем сделано, какие файлы затронуты, какие проверки выполнены и какие ограничения остались.
2. В README нельзя размещать ссылки на репозитории.
3. В README нельзя упоминать похожие чужие программы.
4. После завершения каждой задачи нужно предлагать тематическое улучшение проекта.
5. Проект считается серьезным полноценным продуктом: изменения должны быть внимательными, проверяемыми и сопровождаться логированием проблем.
6. Ошибки и диагностические события приложения должны попадать в лог-файл `logs/volchay-voicer.log`.
7. Интерфейс должен использовать Qt6 QML и SVG-иконки там, где это улучшает читаемость и качество результата.

## Возможности первой версии

- Выбор микрофона и устройства вывода.
- Запуск и остановка real-time аудиопотока.
- Пресеты голоса: чистый, низкий, роботизированный, радио, монстр.
- DSP-цепочка: pitch/formant, robot ring modulation, radio EQ, echo/reverb-like delay, noise gate, compressor/limiter, output gain.
- Панель диагностики: уровень входа, уровень выхода, CPU load, latency estimate, статус потока и последние строки лога.
- Portable ZIP сборка через GitHub Actions для Windows x64.

## Архитектура

- `AudioEngineController` - Qt/QML фасад для управления аудиопотоком.
- `AudioDeviceModel` - модель доступных устройств PortAudio для QML ComboBox.
- `PresetModel` - модель встроенных пресетов и сериализация параметров в JSON.
- `LogController` - чтение последних строк лог-файла для QML.
- `AudioEngine` - PortAudio duplex stream: input callback -> DSP chain -> output callback.
- `EffectsChain` - последовательная обработка эффектов через общий интерфейс `AudioEffect`.
- `PitchShiftEffect` - использует Rubber Band LiveShifter, а архитектурно занимает тот же слот, куда позже можно добавить AI-процессор.

## Сборка

Локально требуется CMake, C++20 compiler, Qt 6.8 LTS или новее, vcpkg и Ninja.

```powershell
cmake --preset windows-release
cmake --build --preset windows-release
ctest --preset windows-release --output-on-failure
```

Если локальная среда не настроена, основная проверка выполняется в GitHub Actions. Workflow собирает Release, запускает тесты, выполняет `windeployqt` и публикует portable ZIP artifact.

## Логи

При запуске создается папка `logs`. Основной файл: `logs/volchay-voicer.log`.

В лог пишутся:

- версия приложения и путь запуска;
- найденные аудиоустройства;
- параметры выбранного потока;
- ошибки PortAudio, DSP и QML backend;
- события Start/Stop;
- последние критичные сообщения для диагностики.

Пользовательский звук по умолчанию не сохраняется.

## Журнал разработки

### 2026-05-24 - Инициализация полноценного проекта

Сформирован план проекта: Windows x64 desktop-приложение на C++ с Qt6 QML, real-time маршрутом микрофон -> DSP -> output device, portable ZIP сборкой и логированием. Выбран классический DSP для первой версии, потому что он дает низкую задержку и не мешает позже подключить AI-модуль отдельным процессором в `EffectsChain`.

Создается структура исходников: отдельные слои для аудиоустройств, аудиодвижка, эффектов, пресетов, логов, QML-интерфейса, SVG-ресурсов и тестов. Это нужно, чтобы UI не зависел напрямую от PortAudio callback, а будущие эффекты можно было добавлять без переписывания окна приложения.

Добавляется CMake/vcpkg/GitHub Actions инфраструктура. Она нужна, чтобы сборка была воспроизводимой, а готовый Windows artifact появлялся после push без ручной упаковки.

### 2026-05-24 - Реализация ядра

Добавлен `PortAudioRuntime` с референсным управлением жизненным циклом PortAudio. Это дает один общий startup/shutdown для моделей устройств и аудиодвижка и не оставляет библиотеку в случайном состоянии.

Добавлены `AudioDeviceModel`, `PresetModel`, `AppSettings`, `LogController` и `AudioEngineController`. Эти классы образуют публичный фасад для QML: список устройств, пресеты, параметры потока, состояние стрима и быстрый доступ к логам.

Реализован `EffectsChain` и набор DSP-эффектов: pitch/formant shift через Rubber Band LiveShifter, ring modulation для robot-режима, радио-эквализация, echo/delay, noise gate и compressor/limiter с output gain. Пока это классический DSP-слой, но архитектурно он уже принимает настройки как отдельный блок, поэтому AI-процессор можно будет вставить позже без ломки UI.

Добавлен QML-интерфейс с двумя рабочими областями: пресеты и аудио-настройки, а ниже диагностика и лог. Также добавлены SVG-иконки для ключевых состояний, чтобы интерфейс оставался читаемым без лишней декоративности.

### 2026-05-24 - Сборка и доставка

Создан workflow для Windows x64 portable ZIP: установка Qt 6.8.3, CMake/Ninja/MSVC, vcpkg-зависимости, сборка Release, тесты, `windeployqt` и архив с бинарником, QML и runtime DLL. Это и есть основной путь доставки результата пользователю.

Созданы тесты для DSP-утилит и пресетов. Они проверяют, что базовая математика не ломается, параметры сериализуются обратно в тот же вид, а встроенные пресеты действительно доступны модели.

### 2026-05-24 - Локальная доводка и проверка

Локально поднят Qt 6.8.3 в пользовательскую папку через `aqtinstall`, после чего `vcpkg` был запущен в manifest mode из собственного `.deps/vcpkg`. Это нужно было, чтобы не зависеть от глобальной системы и повторить поведение CI максимально близко.

Сборка сначала упёрлась в типовые ошибки C++-интеграции: старую сигнатуру PortAudio callback, конфликт имени `PaStream`, неполный тип `RubberBandLiveShifter` и отсутствие runtime DLL в `ctest`. Все эти места были доведены до состояния, где `cmake --build` проходит, а `ctest` стартует тестовый exe с корректным PATH для Qt и vcpkg DLL.

Проверка завершена на локальной машине: Release build собран, тесты проходят, workflow для GitHub Actions остаётся основным способом собрать portable ZIP artifact после push.
